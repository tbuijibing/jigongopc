## 🤖 JiGongOpc とは？

**1人 + AIスタッフのチーム = 1社の会社**

JiGongOpc は「ワンパーソンカンパニー」向けに作られた国内初のAI従業員管理プラットフォームです。1人で会社を運営でき、AIを採用してタスクを割り当て、予算を管理し、意思決定を承認できます。

見た目はタスク管理ツールですが、実際は完全な会社運営システムです：組織構造、財務管理、承認ワークフロー、目標分解、AI従業員の調整がすべて備わっています。

**コードコミットではなく、ビジネス目標を管理します。**

| | 方法 | 例 |
| ------ | --------------- | --------------------------------------------------------------- |
| **01** | 目標を設定 | *「AIメモアプリを作り、3か月で月商100万ドル達成する」* |
| **02** | チームを組む | CEO、CTO、プログラマー、デザイナー、マーケティング — 好きなAIを選べます。 |
| **03** | 実行 | 戦略を承認し、予算を設定し、開始をクリックし、ダッシュボードで監視。 |

## 対象ユーザー

- ✅ **ワンパーソン創業者** — AIでビジネス運営を自動化したい
- ✅ **複数AIユーザー** — OpenClaw、Codex、Claude、Cursorを同時に使い、調整が必要
- ✅ **タブだらけの人** — Claude Codeのウィンドウが20個開いていて、誰が何をしているかわからない
- ✅ **手放し運用派** — AIを24時間稼働させたいが、いつでも介入できる
- ✅ **予算意識の高い人** — コストを監視し、支出上限を設定したい
- ✅ **モバイル勤務派** — スマホから会社を管理したい

## 違いは何か？

| JiGongOpc なし | JiGongOpc あり |
| -------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------- |
| ❌ Claude Codeのタブが20個開いていて、再起動後に全部失われる。 | ✅ タスクベースのチケット、再起動後もセッションが保持されます。 |
| ❌ コンテキストを手動で収集し、AIに何をしているか伝える必要がある。 | ✅ コンテキストがプロジェクト目標を通じて上流へ流れる — AIは常に何を、なぜ行うかを知っています。 |
| ❌ AI設定があちこちに散らばり、車輪の再発明をしている。 | ✅ 組織構造、チケット、委任がすぐに使えます — スクリプトではなく会社を管理します。 |
| ❌ クォータが尽きるまでお金を燃やしていることに気づかない。 | ✅ コスト追跡 + 予算管理、超過時は自動的にスロットリング。 |
| ❌ 定期的なタスク（サポート、SNS、レポート）を毎回手動で開始。 | ✅ ハートビートメカニズムが定期的な作業をスケジュール通りに処理。 |
| ❌ アイデアがあっても、リポジトリを探し、Claude Codeを開き、常に監視する必要がある。 | ✅ JiGongOpcでタスクを追加すれば、AIが自動的に完了し、結果をレビューします。 |

## 技術的ハイライト

| | |
| --------------------------------- | ---------------------------------------------------------------------------------------------- |
| **アトミック実行** | タスクチェックアウトと予算実行はアトミック — 重複作業なし、コスト暴走なし。 |
| **永続状態** | AI従業員がハートビートをまたいで同じコンテキストを復元、最初からやり直しません。 |
| **ランタイム学習** | AIはランタイムでワークフローとプロジェクトコンテキストを学習でき、再トレーニング不要。 |
| **ロールバック可能なガバナンス** | 承認ゲートの強制実行、設定変更のバージョン管理、ミスはロールバック可能。 |
| **目標認識** | タスクに完全な目標チェーンを持ち、AIは常に「なぜ」を知っています。 |
| **会社テンプレート** | 組織、AI従業員、スキルのエクスポート/インポート、自動シークレットサニタイズ。 |
| **複数会社分離** | すべてのエンティティに会社スコープ、1回のデプロイで複数会社、データ完全分離。 |

## 何ではないか？

| | |
| ---------------------------- | ---------------------------------------------------------------------------------------------- |
| **チャットボットではない** | AI従業員には職位があり、チャットウィンドウではありません。 |
| **AIフレームワークではない** | AIの作り方を教えるのではなく、AIを使って会社を運営する方法を教えます。 |
| **ドラッグアンドドロップワークフローではない** | ビジュアルパイプラインはありません。JiGongOpc は実際の会社をモデル化します — 組織構造、目標、予算、ガバナンス。 |
| **プロンプトマネージャーではない** | AIは独自のプロンプト、モデル、ランタイムを持ちます。JiGongOpc はそれらが属する組織を管理します。 |
| **単一AIツールではない** | チーム向けに設計。AIが1つだけならJiGongOpc は必要ないかもしれません。20つあれば絶対に必要です。 |
| **コードレビューツールではない** | JiGongOpc は作業をオーケストレーションしますが、Pull Request ではありません。コードレビューは独自のプロセスです。 |

## クイックスタート

オープンソース。セルフホスト。アカウント不要。

```bash
git clone https://github.com/tbuijibing/jigongopc.git
cd jigongopc
pnpm install
pnpm dev
```

その後、`http://localhost:3100`にアクセス。埋め込みPostgreSQLが自動作成され、追加設定は不要です。

ワンライナーで開始：

```bash
pnpm jigong run
```

> **要件：** Node.js 20+、pnpm 9+

## FAQ

**デプロイ方法は？**
ローカル開発：単一Node.jsプロセスで埋め込みPostgresとローカルファイルストレージを管理。本番：独自のPostgresに接続し、必要に応じてデプロイ。

**複数の会社は？**
はい。1回のデプロイで無限の会社、完全なデータ分離。複数のワンパーソンビジネスを運営する起業家に最適です。

**OpenClaw、Claude Codeとの違いは？**
JiGongOpc はこれらのAIを _使用_ し、組織構造、予算、目標、ガバナンス、説明責任を持つ会社としてオーケストレーションします。

**AIは常時稼働していますか？**
デフォルトのハートビートメカニズムはスケジュール通りに稼働します（タスク割り当て、@メンションでトリガー）。OpenClawのような常時稼働AIにも接続できます。AIを持ってきて、JiGongOpc が調整します。

## 開発

```bash
pnpm dev              # 完全開発モード（API + UI）
pnpm dev:once         # 開発モード（ファイル監視なし）
pnpm dev:server       # サーバーのみ
pnpm build            # すべてのパッケージをビルド
pnpm typecheck        # 型チェック
pnpm test:run         # テスト実行
pnpm db:generate      # DBマイグレーション生成
pnpm db:migrate       # マイグレーション実行
```
