# S400-Template-Marketplace: 公司运营模板与创作者激励系统 — Requirements

> 总工时: 25d | 状态: 🟡 进行中

## 概述

建立 JiGongOpc 公司运营模板商店系统，支持模板创作者发布免费/付费模板，用户通过 CLI 和 Web 界面便捷安装，并通过多级收益分成机制激励创作者持续优化模板。所有模板支持谱系追踪，分叉模板产生收益时自动执行收益分配。

参考: SPEC-400-公司运营模板与创作者激励系统

---

## 用户故事

### US-1: 模板定义与创建
**作为** 模板创作者，**我希望** 按照标准化结构定义公司运营模板，**以便** 其他公司可以复用我的管理规范和流程。

**EARS 验收标准：**
- WHEN 创作者创建模板 THEN 系统应支持定义：角色集合（Roles）、工作流程（Workflows）、交付物规范（Deliverables）、追溯矩阵（Traceability）、复盘规则（Retrospective）
- WHEN 模板被保存 THEN 系统应将模板内容存储为 JSONB 格式，支持加密字段
- WHEN 模板被查询 THEN 系统应在 100ms 内返回完整模板数据
- WHEN 模板内容超过 1MB THEN 系统应自动分片存储并压缩

### US-2: 模板加密与保护
**作为** 模板创作者，**我希望** 保护核心模板逻辑不被轻易复制，**以便** 既能保护知识产权又允许购买者在导入后进行必要的定制。

**EARS 验收标准：**
- WHEN 创作者发布付费模板 THEN 系统应支持两层结构：核心层（Core）AES-256 加密不可修改、定制层（Customization）明文存储允许修改
- WHEN 模板被导出 THEN 系统应生成包含加密核心层和明文定制层的包
- WHEN 模板被导入其他公司 THEN 系统应解密核心层并合并定制层
- WHEN 购买者尝试修改核心层 THEN 系统应拒绝操作并提示"核心逻辑不可修改"

### US-3: 模板分叉与谱系
**作为** 模板创作者，**我希望** 基于现有模板创建改进版本并保留关联关系，**以便** 在原模板产生收益时我能获得分成。

**EARS 验收标准：**
- WHEN 用户分叉模板 THEN 系统应记录完整谱系信息：根模板 ID、父模板 ID、代数、分叉时间和贡献描述
- WHEN 分叉模板产生收益 THEN 系统应按以下比例分配：平台 15%、直接创作者 60%、父模板创作者 15%、根模板创作者 10%
- WHEN 模板没有父模板 THEN 父模板份额应分配给直接创作者
- WHEN 用户查看模板详情 THEN 系统应展示完整的谱系树

### US-4: 免费模板发布
**作为** 模板创作者，**我希望** 免费发布模板以建立声誉，**以便** 为后续付费模板积累受众。

**EARS 验收标准：**
- WHEN 创作者选择"免费发布" THEN 系统应立即发布模板，无需支付审核费用
- WHEN 用户安装免费模板 THEN 系统应无任何费用产生
- WHEN 免费模板被安装 THEN 系统应记录安装统计用于创作者声誉积累
- WHEN 免费模板被分叉 THEN 系统应正常记录谱系，但无收益分配（除非分叉后的模板付费）

### US-5: 付费模板发布与购买
**作为** 模板创作者，**我希望** 为高质量模板设定价格，**以便** 在用户购买时获得收益。

**EARS 验收标准：**
- WHEN 创作者设置模板价格 THEN 系统应支持多币种定价（USD、CNY、JPY 等）
- WHEN 用户购买付费模板 THEN 系统应支持多种支付方式：账户余额、信用卡（Stripe）、支付宝、微信支付
- WHEN 支付完成 THEN 系统应立即解锁模板并允许安装
- WHEN 购买完成 THEN 系统应在 1 秒内完成收益分配计算并更新各创作者账户余额
- WHEN 用户请求退款（7天内） THEN 系统应撤销收益分配并退款

### US-6: 模板搜索发现
**作为** 模板用户，**我希望** 能够方便地发现和搜索需要的模板，**以便** 快速找到适合公司的运营规范。

**EARS 验收标准：**
- WHEN 用户访问模板商店首页 THEN 系统应展示：热门模板、精选付费模板、最新发布的模板、分类浏览
- WHEN 用户输入搜索关键词 THEN 系统应在 200ms 内返回匹配结果
- WHEN 用户按分类筛选 THEN 系统应支持多级分类（行业、规模、用途）
- WHEN 用户按价格筛选 THEN 系统应支持"仅看免费"、"价格区间"等选项
- WHEN 用户按评分排序 THEN 系统应按平均评分从高到低展示

### US-7: CLI 模板安装
**作为** 开发者，**我希望** 通过命令行一键安装模板，**以便** 像使用 npm install 一样简单快捷。

**EARS 验收标准：**
- WHEN 用户运行 `jigong template install owner/template-name` THEN 系统应在 3 秒内完成免费模板安装
- WHEN 用户安装付费模板 THEN 系统应先检查是否已购买，未购买则提示购买流程，已购买则直接下载安装
- WHEN 用户运行 `jigong template search keyword` THEN 系统应返回匹配的模板列表
- WHEN 用户运行 `jigong template preview owner/template-name` THEN 系统应展示模板预览信息
- WHEN 用户运行 `jigong template upgrade owner/template-name` THEN 系统应升级模板并保留定制层内容

### US-8: Web 模板安装
**作为** 非技术用户，**我希望** 通过 Web 界面一键安装模板，**以便** 操作像 App Store 一样简单。

**EARS 验收标准：**
- WHEN 用户在 Web 界面浏览模板 THEN 每个模板应显示"预览"和"安装"按钮
- WHEN 用户点击免费模板的"安装" THEN 系统应立即开始安装并显示进度
- WHEN 用户点击付费模板的"购买并安装" THEN 系统应显示支付弹窗，支持账户余额和第三方支付，支付完成后自动开始安装
- WHEN 安装完成 THEN 系统应显示"安装成功"并引导用户激活模板

### US-9: 收益分配计算
**作为** 平台运营者，**我希望** 每次模板购买时自动执行收益分配，**以便** 创作者及时获得回报。

**EARS 验收标准：**
- WHEN 付费模板被购买 THEN 系统应在 1 秒内完成收益分配计算
- WHEN 收益分配执行 THEN 各参与方应按比例获得收益：平台 15%、直接创作者 60%、父模板创作者 15%、根模板创作者 10%
- WHEN 收益分配完成 THEN 系统应更新各创作者的账户余额并记录交易明细
- WHEN 创作者查看收益报告 THEN 系统应展示：总收益、可提现余额、待结算金额、按模板/时间段汇分的收益

### US-10: 创作者等级系统
**作为** 模板创作者，**我希望** 通过持续创作获得等级提升，**以便** 享受更多平台权益。

**EARS 验收标准：**
- WHEN 创作者满足升级条件 THEN 系统应自动提升其等级（Bronze → Silver → Gold → Platinum → Diamond）
- WHEN 创作者等级提升 THEN 系统应通知创作者并展示新权益（收益加成、平台费减免、专属客服、治理权等）
- WHEN 创作者查看个人中心 THEN 系统应展示当前等级和升级进度
- WHEN Diamond 等级创作者参与平台治理 THEN 系统应允许其对新模板上架进行投票

### US-11: 模板版本管理
**作为** 模板用户，**我希望** 安装特定版本的模板并在新版本发布时选择是否升级，**以便** 控制模板更新节奏。

**EARS 验收标准：**
- WHEN 创作者发布模板 THEN 系统应支持 Semver 版本号（如 1.2.3）
- WHEN 用户安装模板 THEN 系统默认安装最新稳定版本
- WHEN 用户指定版本号 THEN 系统应安装该特定版本
- WHEN 创作者发布新版本 THEN 系统应通知已购买的用户
- WHEN 用户升级模板 THEN 系统应保留定制层的内容

### US-12: 模板可追溯性
**作为** 项目管理者，**我希望** 在使用模板创建的项目中追溯从需求到代码的完整链路，**以便** 事后能查原因。

**EARS 验收标准：**
- WHEN 项目使用模板创建 THEN 系统应自动生成追溯矩阵
- WHEN 事务被创建 THEN 系统应自动关联相关的 spec 文档
- WHEN 代码被提交 THEN 系统应自动关联对应的事务和需求
- WHEN 用户查看事务详情 THEN 系统应展示完整的追溯链路：需求 → 设计 → 实现 → 测试

### US-13: 复盘与自我进化
**作为** 模板创作者，**我希望** 了解模板在实际使用中的表现，**以便** 持续改进。

**EARS 验收标准：**
- WHEN 基于模板的事务完成 THEN 系统应自动生成复盘报告
- WHEN 复盘报告生成 THEN 系统应分析：流程执行效率、角色工作负载、交付物质量、与预期的偏差
- WHEN 多个复盘数据积累 THEN 系统应识别优化模式并向创作者提供改进建议
- WHEN 创作者采纳建议 THEN 系统应记录改进效果用于模型训练

### US-14: 跨公司模板复制
**作为** 公司管理员，**我希望** 将其他公司的优秀模板导入到我们公司使用，**以便** 借鉴行业最佳实践。

**EARS 验收标准：**
- WHEN 模板被导出 THEN 系统应生成可移植的模板包
- WHEN 模板包被导入新公司 THEN 系统应解密核心层、保留定制层供修改、建立与原始模板的谱系关联
- WHEN 导入的模板产生收益 THEN 原始创作者应获得相应分成

### US-15: 收益提现
**作为** 模板创作者，**我希望** 将收益提现到银行账户或第三方支付账户，**以便** 获得实际回报。

**EARS 验收标准：**
- WHEN 创作者申请提现 THEN 系统应验证可提现余额 ≥ $100
- WHEN 提现申请提交 THEN 系统应进入审核流程（自动或人工）
- WHEN 提现完成 THEN 系统应扣除相应余额并记录交易
- WHEN 创作者绑定提现方式 THEN 系统应支持银行账户、PayPal、支付宝等多种方式

## 非功能需求

| 维度 | 要求 |
|---|---|
| 响应时间 | 模板查询 < 100ms，商店搜索 < 200ms，收益分配计算 < 1s |
| 模板安装 | 免费模板 < 3s，付费模板（含支付）< 10s |
| 加密强度 | 核心层使用 AES-256-GCM |
| 支付安全 | PCI DSS 合规 |
| 权限控制 | RBAC + 数据范围 |
| 可用性 | 模板商店 99.9% 可用性 |
| 国际化 | 支持多语言模板描述（参考 SPEC-102） |
| 时区 | 收益统计按 UTC 存储，展示按用户时区（参考 SPEC-109） |
