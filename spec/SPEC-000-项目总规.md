## 一、项目概述

### 1.1 项目名称

- **中文名称**：济公 OPC-Java
- **英文名称**：JiGongOpc-Java
- **项目代号**：JGO-Java

### 1.2 项目定位

JiGongOpc-Java 是一个 **AI Agent 编排控制平面系统**，提供多公司/多租户管理能力，支持用户创建和管理多个 AI 公司，每个公司拥有独立的 Agent 团队、任务系统、目标管理、成本预算和审批治理。

### 1.3 核心价值

- **多公司管理**：公司作为数据隔离的顶层单元，通过租户 ID 实现数据隔离
- **Agent 编排**：支持多种适配器（Process/HTTP/Gateway）调用 AI Agent
- **任务调度**：基于 XXL-Job 的心跳调度机制，实时检取原子任务
- **目标驱动**：Goal/Project/Issue 三层目标管理体系
- **成本管控**：月度预算、实时成本跟踪、预算利用率分析
- **审批治理**：基于工作流引擎的审批流和治理规则

### 1.4 设计原则

1. **目的优先**：每个页面有明确的单一目的
2. **层次分明**：一个主导视觉区域，次要区域从属
3. **渐进披露**：逐步展示复杂度
4. **状态可见**：系统状态始终可见
5. **反馈及时**：每个操作都有明确反馈
6. **表面问题，不隐藏**：审计和可见性优先，不自动恢复，让人类或管理 Agent 决定如何处理
7. **原子所有权**：单一分配者 per 任务，原子检出防止冲突
8. **任务即通信**：所有 Agent 通信通过任务 + 评论，无侧信道
9. **不限制 Agent 实现**：任何语言、任何框架、任何运行时

## 七、文档规范

### 7.1 Spec 文档结构

每个 Spec 模块包含 5 个文档：

1. **需求文档** (`requirements.md`) - EARS 格式的用户故事和验收标准
2. **技术设计文档** (`design.md`) - 分层架构、数据库设计、API 设计
3. **UI/UX 设计文档** (`SPEC-xxx-3-UI-UX-设计文档.md`) - Pencil 设计图关联
4. **任务文档** (`tasks.md`) - 任务分解和工时估算
5. **验收文档** (`SPEC-xxx-5-验收文档.md`) - 功能和性能验收用例

### 7.2 文档格式

- **需求文档**：采用 EARS 格式（When/Then）
- **技术设计**：包含分层架构图、数据库设计、API 设计
- **UI/UX**：Pencil 设计图 ID 与前端页面、后端 API 关联
- **任务文档**：任务拆解、依赖关系、工时估算
- **验收文档**：功能验收、性能验收、安全验收

## 八、明确不做的事情（Anti-Requirements）

JiGongOpc-Java 明确**不**做的事情：

| 不做的事情 | 说明 | 替代方案 |
|------------|------|----------|
| **不是 Agent 运行时** | 只编排，不执行 | Agent 在外部运行 |
| **不包含知识库** | 核心无 Wiki/文档/向量库 | 未来作为插件 |
| **不是 SaaS** | 单租户、自托管 | 本地/私有化部署 |
| **不限制 Agent 实现** | 任何语言/框架/运行时 | 适配器模式支持 |
| **不自动自愈** | 表面问题，不自动恢复 | 通过仪表板暴露 |
| **不管理工作产出物** | 无代码库/文件系统/部署管理 | 只登记产出物链接 |
| **不自动重新分配任务** | 停滞任务通过仪表板暴露 | 人类或管理 Agent 决定 |
| **不追踪外部收支** | Token/LLM 成本是核心 | 外部收支作为插件 |

## 九、插件扩展架构

### 9.1 插件设计原则

核心系统必须可扩展，以下功能作为**插件**实现，无需修改核心：

| 插件类型 | 说明 | 实现方式 |
|----------|------|----------|
| **Agent 适配器插件** | 新的 Adapter 类型（如 Claude Code、LangChain） | 实现 AgentAdapter 接口 |
| **知识基地插件** | 文档/Wiki/向量数据库集成 | 独立服务 + API 集成 |
| **外部收支插件** | 外部收入/支出跟踪 | 扩展现有成本模块 |
| **通知插件** | Slack/邮件/钉钉通知 | 事件钩子 + 通知服务 |
| **报表插件** | 自定义报表生成 | 报表引擎 + 导出服务 |

### 9.2 扩展点

- **事件系统**：任务/Agent 生命周期事件钩子
- **API 边界**：清晰定义的 API 接口供插件调用
- **UI 组件**：（未来）可注册的 UI 组件
