## 🤖 JiGongOpc 是啥？

**一个人 + 一群 AI 员工 = 一家公司**

JiGongOpc 是国内首个给「一人公司」打造的 AI 员工管理平台。你一个人就能开公司，招 AI 当员工、派任务、管预算、审决策，全在一个地方搞定。

别被它的外表骗了 —— 看着像个任务管理器，实际上是个完整的公司运营系统：组织架构、财务管控、审批流程、目标拆解、AI 员工协调，全都有。

**管的是业务目标，不是代码提交。**

| | 怎么做 | 举个例子 |
| ------ | --------------- | --------------------------------------------------------------- |
| **01** | 定目标 | *"做个 AI 笔记应用，三个月做到月入百万美金。"* |
| **02** | 组团队 | CEO、CTO、程序员、设计师、市场 —— 想用哪个 AI 就用哪个。 |
| **03** | 开干 | 审批策略、设定预算、点击启动、看面板监控。 |

## 有啥不一样？

| 没用 JiGongOpc | 用了 JiGongOpc |
| -------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------- |
| ❌ 开了 20 个 Claude Code 标签页，重启后全丢。 | ✅ 任务基于工单，会话跨重启持久化。 |
| ❌ 手动到处找上下文，提醒 AI 你在干啥。 | ✅ 上下文从任务向上流经项目目标 — AI 始终知道干啥、为啥。 |
| ❌ AI 配置文件散落各处，自己在造轮子。 | ✅ 组织架构、工单、委派开箱即用 —— 你管的是公司，不是脚本。 |
| ❌ 烧钱烧到配额耗尽才发现。 | ✅ 成本追踪 + 预算管控，超额自动限流。 |
| ❌ 定期任务（客服、社交、报告）每次都手动启动。 | ✅ heartbeat 机制按计划处理常规工作。 |
| ❌ 有个想法得找仓库、开 Claude Code、全程盯着。 | ✅ 在 JiGongOpc 加个任务，AI 自动完成，你审核结果。 |

## 技术亮点

| | |
| --------------------------------- | ---------------------------------------------------------------------------------------------- |
| **原子化执行** | 任务签出和预算执行是原子操作，不重复工作，不烧钱失控。 |
| **持久化状态** | AI 员工跨心跳恢复相同上下文，不用从头开始。 |
| **运行时学习** | AI 能在运行时学习工作流和项目上下文，不用重新训练。 |
| **可回滚治理** | 审批门控强制执行，配置变更有版本，错了能回滚。 |
| **目标感知** | 任务携带完整目标链路，AI 始终知道"为啥"。 |
| **公司模板** | 导出/导入组织、AI 员工和技能，自动处理密钥清洗。 |
| **多公司隔离** | 每个实体有公司作用域，一次部署多家公司，数据完全隔离。 |

## 不是什么？

| | |
| ---------------------------- | ---------------------------------------------------------------------------------------------- |
| **不是聊天机器人** | AI 员工有工作岗位，不是聊天窗口。 |
| **不是 AI 框架** | 不教你怎么造 AI，教你怎么用 AI 运营公司。 |
| **不是拖拽式工作流** | 没有拖拽流水线。JiGongOpc 建模的是真实公司 —— 有组织架构、目标、预算和治理。 |
| **不是提示词管理器** | AI 自带提示词、模型和运行时。JiGongOpc 管的是它们所在的组织。 |
| **不是单 AI 工具** | 为团队设计。只有一个 AI 可能不需要 JiGongOpc，有二十个就一定需要。 |
| **不是代码审查工具** | JiGongOpc 编排工作，不是 Pull Request。代码审查流程自己搞定。 |

## 快速开始

开源。自托管。无需账号。

```bash
git clone https://github.com/tbuijibing/jigongopc.git
cd jigongopc
pnpm install
pnpm dev
```

启动后访问 `http://localhost:3100`。内嵌 PostgreSQL 自动创建，无需额外配置。

一键启动：

```bash
pnpm jigong run
```

> **环境要求：** Node.js 20+，pnpm 9+

## FAQ

**咋部署？**
本地开发时，一个 Node.js 进程管内嵌 Postgres 和本地文件存储。生产环境可以连你自己的 Postgres，按需部署。

**能跑多家公司吗？**
能。单次部署无限公司，数据完全隔离。适合同时运营多个一人公司的创业者。

**和 OpenClaw、Claude Code 啥区别？**
JiGongOpc _用_ 这些 AI，把它们编排成一家公司 —— 有组织架构、预算、目标、治理和问责。

**AI 是一直跑着的吗？**
默认 heartbeat 机制按计划跑（任务分配、@提及触发）。也能接 OpenClaw 等持续运行的 AI。你提供 AI，JiGongOpc 负责协调。

## 开发

```bash
pnpm dev              # 完整开发模式（API + UI）
pnpm dev:once         # 开发模式（不监听文件变更）
pnpm dev:server       # 仅服务端
pnpm build            # 构建所有包
pnpm typecheck        # 类型检查
pnpm test:run         # 运行测试
pnpm db:generate      # 生成数据库迁移
pnpm db:migrate       # 执行迁移
```
